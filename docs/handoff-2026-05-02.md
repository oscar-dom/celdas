## Para sincronizar un PC nuevo

### 1. Instalar herramientas (una vez)

- **Git** y **VS Code** (probablemente ya están)
- **GitHub CLI:** `winget install --id GitHub.cli -s winget` (Windows) o `brew install gh` (Mac)
- **Node.js LTS (20+):** `winget install --id OpenJS.NodeJS.LTS -s winget`
- **pnpm:**
  - Windows: primero `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` (acepta con `Y`), luego `npm install -g pnpm`
  - Mac/Linux: `npm install -g pnpm`

### 2. Autenticar y clonar

```powershell
gh auth login                                      # se abre el navegador
gh repo clone oscar-dom/celdas <ruta>\celdas
gh repo clone oscar-dom/claude-config <ruta>\claude-config
```

### 3. Bootstrap del proyecto

```powershell
cd <ruta>\celdas
git checkout develop && git pull                   # o la rama del PR si aún no se mergeó
pnpm install                                       # instala deps + activa hooks de husky
pnpm dev                                           # debe arrancar en http://localhost:3000
```

## Gotchas conocidos (no caigas otra vez)

### pnpm 10 + workspace.yaml

`pnpm add <paquete>` falla con `Cannot destructure property 'manifest'...` por un bug de pnpm 10. **Workaround obligatorio**: usar `pnpm add --ignore-workspace <paquete>` para añadir cualquier dependencia nueva. Está en `CLAUDE.md`.

### Husky desde git-bash en Windows

Los hooks (`pre-commit`, `commit-msg`) ejecutan `pnpm`, pero **git-bash en Windows hereda un PATH limitado** que no incluye `C:\Users\<usuario>\AppData\Roaming\npm`. Resultado: el hook falla con "pnpm: command not found".

**Workaround:** commitea desde la **terminal integrada de VS Code** (PowerShell, no git-bash) o desde una **PowerShell normal**. Esos sí tienen Node y pnpm en PATH y los hooks funcionan.

### AGENTS.md generado por Next.js 16

`AGENTS.md` en el root contiene avisos para agentes de IA sobre los breaking changes de Next 16. Está importado desde `CLAUDE.md` con la línea `@AGENTS.md`. **No lo borres ni quites el import.**

### shadcn `form` no portado a `base-nova`

El componente `form` de shadcn aún no existe en el style `base-nova` (que es el que usamos). Cuando llegue el momento de hacer login/register en Fase 1, opciones:

1. Cambiar style del proyecto a `default` o `new-york` (rebuilds todos los componentes)
2. Copiar el `form.tsx` de un proyecto con `new-york` y adaptarlo a `@base-ui/react`
3. Usar un registry custom
